# 播放器与弹幕系统架构说明

## 一、系统概述

本项目的视频播放器是一个高度定制化的 HTML5 播放器解决方案，基于 **ArtPlayer** 构建，集成了弹幕系统、自定义控制栏、HLS 流媒体播放等功能。采用分层架构设计，实现播放器核心与弹幕系统的完全解耦。

### 核心特性

- **HLS 流媒体支持**：基于 hls.js，支持自适应码率
- **弹幕系统**：实时弹幕发送、显示、点赞功能
- **自定义控制栏**：完全自定义的 UI 控件
- **多播放源支持**：支持切换播放线路
- **移动端优化**：iOS Safari 全屏兼容、手势支持
- **响应式设计**：PC/移动端自适应布局

## 四、播放器核心架构

### 4.1 VideoPlayer 组件

**核心职责**：
- 初始化 ArtPlayer 实例
- 管理 HLS.js 实例
- 暴露播放控制接口
- 同步播放器状态到父组件

**状态管理**：

| 状态 | 类型 | 说明 |
|------|------|------|
| `isPaused` | boolean | 是否暂停 |
| `isMuted` | boolean | 是否静音 |
| `volume` | number | 音量（0-1） |
| `currentTime` | number | 当前播放时间 |
| `duration` | number | 视频总时长 |
| `buffered` | number | 已缓冲时间 |
| `isFullscreen` | boolean | 是否全屏 |
| `currentPlaybackRate` | number | 当前播放速率 |
| `currentAspectRatio` | string | 当前画面比例 |

**暴露接口**：

| 方法 | 功能 |
|------|------|
| `play()` | 播放 |
| `pause()` | 暂停 |
| `seek(time)` | 跳转到指定时间 |
| `toggleFullscreen()` | 全屏切换 |
| `toggleMute()` | 静音切换 |
| `setVolume(vol)` | 设置音量 |
| `setPlaybackRate(rate)` | 设置播放速率 |
| `setAspectRatio(ratio)` | 设置画面比例 |
| `setHasMultipleSources(value)` | 设置多源标记 |

### 4.2 ArtPlayerAdapter 适配器

采用适配器模式封装 ArtPlayer，实现播放器实例的统一接口：

| 方法 | 功能 |
|------|------|
| `play()` | 播放 |
| `pause()` | 暂停 |
| `seek(time)` | 跳转 |
| `destroy()` | 销毁实例 |
| `on(event, callback)` | 注册事件监听 |
| `off(event, callback)` | 移除事件监听 |

**HLS.js 配置优化**：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `maxBufferLength` | 30 | 最大缓冲秒数 |
| `maxMaxBufferLength` | 60 | 绝对最大缓冲秒数 |
| `maxBufferSize` | 60MB | 最大缓冲大小 |
| `startLevel` | -1 | 自动选择最佳质量 |
| `capLevelToPlayerSize` | true | 根据播放器尺寸限制质量 |
| `capLevelOnFPSDrop` | true | FPS 下降时降级 |
| `fragLoadingMaxRetry` | 6 | 片段重试次数 |
| `manifestLoadingMaxRetry` | 3 | 清单重试次数 |
| `enableWorker` | true | 启用 Web Worker 解码 |
| `enableSoftwareAES` | true | 软件 AES 解密 |

**错误恢复机制**：

| 错误类型 | 处理方式 |
|---------|---------|
| 网络错误 (NETWORK_ERROR) | 延迟后重新加载 |
| 媒体错误 (MEDIA_ERROR) | 尝试恢复 |
| 致命错误 | 无法恢复，发出错误事件 |

### 4.3 全屏控制

兼容多种浏览器和 iOS Safari：

**退出全屏**：
- 调用 `document.exitFullscreen()`
- 更新全屏状态

**进入全屏**：
- iOS Safari：使用 `webkitEnterFullscreen` 特殊处理
- 标准浏览器：调用 `video.requestFullscreen()`
- 移动端：尝试锁定横屏方向

- 弹幕开关控制

| 方法/属性 | 说明 |
|----------|------|
| `items` | 弹幕列表（响应式） |
| `loading` | 加载状态 |
| `error` | 错误信息 |
| `enabled` | 弹幕开关 |
| `loadDanmaku(startTime, endTime)` | 从服务器加载弹幕列表 |

**发送弹幕流程**：

5. 失败则移除临时弹幕并提示错误

| 付费用户 | 金色 (#FFD700) | 中 |
| 普通用户 | 白色 (#FFFFFF) | 默认 |

## 六、弹幕渲染层 (DanmakuLayer)

### 6.1 设计理念

弹幕层完全独立于播放器，通过 `currentTime` prop 同步，实现：
- **解耦**：弹幕渲染不依赖播放器实例
- **性能**：使用 Web Animations API 实现流畅动画
- **灵活**：支持多种弹幕类型和设置

### 6.2 轨道系统

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `trackStates` | number[] | 每个轨道的结束时间 |
| `trackCount` | 8 | 轨道数量 |
| `trackHeight` | 36px | 轨道高度（移动端 28px） |

**碰撞检测逻辑**：
- 遍历所有轨道，查找结束时间早于当前时间的轨道
- 若所有轨道都被占用，随机选择一个轨道

### 6.3 动画系统

使用 **Web Animations API** 实现高性能动画：

**滚动弹幕动画参数**：

| 参数 | 说明 |
|------|------|
| `speed` | 速度（根据设置：80/120/180 像素/秒） |
| `containerWidth` | 容器宽度 |
| `textWidth` | 文本宽度 |
| `distance` | 移动距离（容器宽度 + 文本宽度） |
| `duration` | 动画时长（距离 / 速度） |
| `delay` | 延迟时间（弹幕时间 - 当前时间） |

**速度映射**：

| 级别 | 速度（像素/秒） |
|------|----------------|
| slow | 80 |
| normal | 120 |
| fast | 180 |

### 6.4 弹幕类型

| 类型 | 渲染方式 | 持续时间 |
|------|---------|---------|
| `scroll` | 从右到左滚动 | 根据距离和速度计算 |
| `top` | 顶部固定居中 | 3秒后淡出 |
| `bottom` | 底部固定居中 | 3秒后淡出 |

### 6.5 过滤系统

**被屏蔽的模式列表**：
- 根据设置判断是否屏蔽滚动弹幕
- 根据设置判断是否屏蔽顶部弹幕
- 根据设置判断是否屏蔽底部弹幕

**过滤逻辑**：
- 计算被屏蔽的模式列表
- 过滤弹幕列表，排除被屏蔽模式的弹幕

### 6.6 性能优化

**CSS 优化**：
- 使用 `will-change: transform` 启用 GPU 加速
- 使用 `transform: translateZ(0)` 触发硬件加速
- 设置 `user-select: none` 防止文本选择

**渲染优化**：
- 只渲染当前时间窗口内的弹幕（±3秒）
- 已渲染弹幕去重（使用 Set 存储已渲染ID）
- 定期清理过期弹幕（10秒前）

**动画优化**：
- 使用 Web Animations API（GPU 加速）
- 避免频繁的 DOM 操作
- 动画结束时自动清理 DOM

**状态同步函数**：
- 从播放器实例读取状态
- 更新父组件的响应式状态

## 十一、交互流程

### 11.1 播放流程

```
用户进入播放页
    │
    ▼
解析 slug 获取 movieId
    │
    ▼
获取影片详情 + 剧集列表
    │
    ▼
检查播放权限 (source_limit)
    │
    ├─ 0: 直接播放
    ├─ 1: 未登录则显示登录弹窗
    └─ 2: 显示付费提示
    │
    ▼
初始化播放器 (VideoPlayer)
    │
    ▼
加载弹幕数据
    │
    ▼
开始播放 + 弹幕同步
```

### 11.2 发送弹幕流程

```
用户输入弹幕内容
    │
    ▼
点击发送按钮
    │
    ▼
检查登录状态
    │
    ├─ 未登录: 触发登录弹窗
    └─ 已登录: 继续
    │
    ▼
创建临时弹幕项 (temp_xxx)
    │
    ▼
如果正在播放: 立即显示弹幕
    │
    ▼
调用后端 API
    │
    ▼
成功: 更新临时ID为真实ID + 更新颜色
失败: 移除临时弹幕 + 提示错误
```

### 11.3 弹幕提醒机制

**触发条件**：
- 每集只触发一次
- 弹幕开关已开启
- 用户已登录
- 未来10秒内没有弹幕

**处理方式**：
- 显示本地提醒弹幕（不走接口）
- 标记已触发

## 十二、性能优化总结

### 12.1 播放器优化

| 优化项 | 方法 |
|--------|------|
| HLS 缓冲 | 限制最大缓冲 60MB，避免内存溢出 |
| 码率自适应 | 根据带宽和播放器尺寸自动切换 |
| 错误恢复 | 网络错误自动重试，媒体错误自动恢复 |
| GPU 加速 | CSS transform + will-change |

### 12.2 弹幕优化

| 优化项 | 方法 |
|--------|------|
| 渲染优化 | 只渲染时间窗口内的弹幕 |
| 动画优化 | Web Animations API + GPU 加速 |
| 碰撞检测 | 轨道状态管理，避免重叠 |
| 内存优化 | 动画结束自动清理 DOM |

### 12.3 移动端优化

| 优化项 | 方法 |
|--------|------|
| 全屏兼容 | iOS Safari 使用 webkitEnterFullscreen |
| 横屏锁定 | 全屏时尝试锁定横屏方向 |
| 手势支持 | 长按快进，滑动控制音量/亮度 |
| 触摸优化 | 防止弹幕误触（pointer-events: none） |

## 十三、最佳实践

### 13.1 开发规范

1. **保持解耦**：弹幕层不依赖播放器实例
2. **状态同步**：使用 RAF 同步，避免事件延迟
3. **错误处理**：播放器错误需分类处理（网络/媒体/致命）
4. **内存管理**：组件卸载时清理动画和事件监听

### 13.2 用户体验

1. **实时反馈**：发送弹幕立即显示，失败再移除
2. **自动隐藏**：控制栏 2 秒自动隐藏
3. **弹幕提醒**：弹幕稀少时鼓励用户发送
4. **角色区分**：通过颜色区分用户角色
